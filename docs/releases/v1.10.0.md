# toolu v1.10.0

Released: 2026-06-14

## Highlights

This release adds two segments to the optional `statusline` plugin, both computed off the render hot path:

- **`wk:` — weekly token usage.** Account-wide tokens consumed this ISO week (Mon–Sun, local time): `input + output + cache_creation` (cache_read excluded as ~98% noise that does not pace the rate-limit window), summed across the main agent **and** its subagents, deduped by message id. A `Stop` hook parses the transcript once per turn and writes per-`(week, session)` files; the statusline only sums them, so the render stays cheap. Each Monday a fresh bucket starts and prior weeks are kept on disk for future reporting.
- **`[mem:N]` — comemory count.** When `code-intel` is installed, the statusline shows the project's memory count, published once per session by a new `code-intel` `SessionStart` module. The repo key is derived via `git-common-dir`, so a worktree resolves to its main repo's scope rather than reading zero.

## Upgrade notes

No action needed. Both segments light up automatically when the relevant plugins are installed and degrade silently when they are not. If you already run `statusline`, update it to pick up `wk:` and `[mem:N]`:

```text
/plugin install statusline@toolu
/plugin install code-intel@toolu
```

## Included changes since v1.9.0

- `statusline`: weekly token-usage segment (`wk:`) backed by a `Stop`-hook ledger — deduped by message id, `cache_read` excluded, subagent transcripts included, bucketed by each message's local ISO week, race-free per-`(week, session)` files. (`statusline` 0.2.0 → 0.3.0)
- `statusline`: comemory count segment (`[mem:N]`), worktree-correct key derivation; `format_tokens` gains an `M` tier.
- `code-intel`: new `SessionStart` module that publishes the per-project memory count for the statusline. (`code-intel` 0.2.0 → 0.3.0)
- `code-intel`: test hardening — `comemory-status.bats` teardown is safe when the comemory binary is absent (CI skip path).
